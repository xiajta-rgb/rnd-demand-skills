# 研发需求智能治理技能

一个专为研发部门打造的自动化需求评审工具，基于公司评分规则，实现「一段话描述 → AI 智能评审 → 自动打分 → 定级分类 → 材料生成」的全流程自动化。

## 核心工作流

```
用户输入：一段话描述需求
       ↓
AI 自动提取：痛点/收益/影响/投入/紧急程度
       ↓
AI 智能评分：6维度打分 + 定级 + 决策
       ↓
输出结果：评分报告 + 优先级 + 提报材料
```

---

## 使用方式

### 一句话提交需求（推荐）

直接描述你的需求，AI 自动提取关键信息并评分：

```
你：帮我评估这个需求，我们要做一个亚马逊侵权专利库的自动化采集脚本，
现在手动采集太慢了还容易漏，已经收到两次侵权警告了，法务和运营都在用，
大概1人月能搞定，下个月底前必须上线。

AI：[自动提取] → 自动评分 → 93分 A级 → 独立立项评审
```

### 批量需求评审

一次提交多个需求：

```
你：帮我评审这批需求：

1. 跨部门数据中台，把销售财务供应链数据整合，提升40%效率，3个部门用，3人月，年度战略项目
2. 个人界面按钮改颜色，就我习惯用，无业务价值，半天搞定，不急
3. 客服系统智能回复，减少30%人工量，客服和技术部用，2人月，老板希望尽快上线

AI：[批量评分]
- D001: 88分 A级战略型 → 独立立项
- D002: 18分 D级无效 → 驳回
- D003: 65分 B级经营优化 → 纳入评审
```

### 快速单个评估

```
你：评估：修复登录页偶尔报错的bug，半天搞定，不急

AI：52分 C级日常改进 → 轻量快速处理
```

## AI 自动提取的维度

当你描述需求时，AI 会自动提取以下信息：

| AI 提取维度 | 示例关键词 |
|-------------|-----------|
| 业务痛点 | "太慢"、"容易漏"、"侵权警告"、"客户投诉" |
| 预期收益 | "提升效率"、"减少成本"、"降低差错率" |
| 影响范围 | "法务和运营都在用"、"3个部门"、"全公司" |
| 资源投入 | "1人月"、"半天"、"2人周" |
| 紧急程度 | "下月底必须上线"、"不急"、"有截止日" |
| 战略匹配 | "年度战略项目"、"老板要求的" |

---

## 评分机制

| 维度 | 权重 | 说明 |
|------|------|------|
| 业务价值 | 25分 | 收入提升/降本增效/核心业务贡献 |
| 战略匹配度 | 20分 | 与公司当前战略重点的契合度 |
| 影响范围 | 15分 | 跨部门/多岗位受益的复用价值 |
| 投入产出比 | 15分 | ROI评估，低投入高收益得分高 |
| 紧急程度 | 15分 | 合规/风险/客户影响类高紧急需求 |
| 实施复杂度 | 10分 | 反向计分，技术难度越低得分越高 |

| 总分 | 等级 | 类型 | 决策 |
|------|------|------|------|
| ≥80分 | A级 | 战略型需求 | 独立立项评审 |
| 60-79分 | B级 | 经营优化型需求 | 纳入评审流程 |
| 40-59分 | C级 | 日常改进型需求 | 轻量快速处理 |
| <40分 | D级 | 无效型需求 | 驳回并说明原因 |

- 关键词列表
- 规则条件
